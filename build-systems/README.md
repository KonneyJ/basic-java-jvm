# Управление версиями зависимостей
В проект добавлено 2 библиотеки.  

Библиотека jackson-jaxrs-json-provider:2.13.4 тянет jackson-core, jackson-databind, jackson-annotations версии 2.13.4.

Библиотека jackson-datatype-jsr310:2.15.3 тянет те же артефакты версии 2.15.3.

Версии несовместимы из‑за изменений в API между 2.13 и 2.15.

После выполнения команды mvn dependency:tree обнаружен конфликт версий.  

jackson-jaxrs-json-provider требует jackson-databind:2.13.4, а jackson-datatype-jsr310 требует jackson-databind:2.15.3.

В classpath попадёт только одна версия (Maven выберет по правилу nearest definition).
В данном случае победит 2.15.3, так как она указана в последней зависимости. 

Однако это не гарантирует 
корректной работы jackson-jaxrs-json-provider, который рассчитан на API версии 2.13.4.

Последствия конфликта:

- Возможны NoSuchMethodError при запуске.
- Потеря обратной совместимости (например, изменённые сигнатуры методов).
- Непредсказуемое поведение сериализации/десериализации JSON.

# Жизненный цикл сборки
- mvn clean  
В ходе этого цикла происходит очистка проекта. Удаляются все ранее созданные в процессе сборки артефакты: 
jar-файлы, файлы классов и др.
- mvn compile  
В ходе этого цикла происходит компиляция проекта, без тестов. Создается папка target. В подпапке classes находятся 
скомпилированные классы. В подпапке generated-sources находится автоматически сгенерированный исходных код. В подпапке 
maven-status находится служебная информация о процессе сборки.
- mvn test  
Происходит выполнение тестов с использованием указанного тестового фреймворка. Включает в себя очистку и компиляцию кода.
В консоли указывается информация о пройденных тестах. Генерирует папки аналогичные предыдущему пункту, но для тестов 
проекта.
- mvn package  
Происходит упаковка артефакта, например в jar-файл, который создается в ходе этого цикла. Включает очистку, компиляцию 
и тесты. Также создает папку maven-archiver с файлом pom.properties.
